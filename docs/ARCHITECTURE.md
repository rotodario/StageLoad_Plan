# Architecture

StageLoad Planner 3D is structured as a local-first technical planning app. The current MVP keeps the domain logic independent from the React UI so the project can evolve toward PDF export, auto-pack, collaboration, or a backend without rewriting the geometry core.

## Main Layers

- `src/types`: domain models for trucks, item templates, item instances, plans, load walls and warnings.
- `src/data`: default truck and sample live-production load templates.
- `src/store`: Zustand global state, derived reports, selection, mutations and local persistence.
- `src/utils`: pure logic for units, geometry, collisions, load walls, JSON export and future PDF payloads.
- `src/components/layout`: application shell and fixed CAD-style panels.
- `src/components/viewport`: React Three Fiber scene, truck model, grid, item meshes, transform controls and shortcuts.
- `src/components/library`: item template library and template creation.
- `src/components/inspector`: selected item and truck editing panels.
- `src/components/loadwalls`: load wall summaries and front-view miniatures.
- `src/components/reports`: validation, load order and summary panels.

## Units

The domain model works in millimeters for precision:

```text
1000 mm = 1 meter = 1 Three.js unit
```

Conversion is centralized in `src/utils/units.ts`.

## Geometry Model

Each `LoadItemInstance` references a `LoadItemTemplate`. The template defines real dimensions and weight. The instance defines position, rotation, visibility, lock state, load order and unload priority.

Bounding boxes are generated from the template dimensions plus the instance transform. The MVP supports 90-degree rotation on the vertical axis and uses axis-aligned bounding boxes for collision detection.

## Validation

Current validation checks:

- Total weight against truck maximum.
- Item bounds against truck interior.
- Basic AABB collisions.
- Floating items that are not supported by the floor or another item.
- Basic unload blocking based on X position toward the door.

## Load Walls

Load walls are generated by slicing the truck length into 1200 mm sections. A wall includes any item whose bounding box intersects that slice.

This is intentionally deterministic and simple for the MVP. Later versions can support custom wall depths, manual wall grouping and print-ready layouts.

## Persistence

Current persistence is local:

- `localStorage` for quick save/load.
- JSON export/import for portable plan files.

The store is the boundary where IndexedDB or backend persistence can be added later.

## PDF Export Preparation

`src/utils/exportPdf.ts` defines a `PdfExportPayload` containing the plan, report, generated walls and timestamp. Actual PDF rendering should consume this payload rather than coupling directly to UI components.
